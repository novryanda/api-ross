## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Seed data

Run migrations first, then seed the demo data:

```bash
$ npx prisma migrate deploy
$ npm run db:seed
```

The seed is idempotent and creates demo users, active campaigns, social
accounts, blast targets/attempts/reports, comment commands/tasks, one export,
and sample audit logs. Default password is `Password123!`; override it with
`SEED_DEFAULT_PASSWORD`. Admin credentials can be overridden with
`SEED_ADMIN_EMAIL` and `SEED_ADMIN_NAME`.

Default demo users:

```text
admin@ross.local
buzzer.ayu@ross.local
buzzer.bima@ross.local
buzzer.citra@ross.local
viewer@ross.local
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Export file storage

Campaign exports store generated PDF/XLSX files in a private Cloudflare R2
bucket. The frontend download endpoint stays the same:

```text
GET /api/v1/exports/:exportId/download
```

R2 credentials are provided through `.env` or deployment secrets. Do not commit
real credentials:

```env
R2_ENDPOINT="https://<account_id>.r2.cloudflarestorage.com"
R2_BUCKET="ross-project"
R2_ACCESS_KEY_ID="<access_key_id>"
R2_SECRET_ACCESS_KEY="<secret_access_key>"
R2_PUBLIC_BASE_URL=""
R2_FORCE_PATH_STYLE=true
```

When `R2_PUBLIC_BASE_URL` is empty, files are downloaded by streaming the
private object through the backend endpoint. Private R2 URLs and credentials are
not exposed to clients.

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.
